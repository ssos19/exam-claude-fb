# 요구사항: 축구공 위치 기반 점유율 추적

## 환경
- Node.js, Next.js
- MySQL (mysql2 라이브러리 사용)

## 도메인 규칙
- 공 위치는 1차원(좌/우)으로만 표현한다. 값은 0~100 사이의 정수이며, 0=완전 왼쪽, 100=완전 오른쪽을 의미한다.
- 팀은 두 개다: A팀(Anthro, 별칭 Ant), B팀(Bob, 별칭 Bike)
- 각 팀은 좌/우 중 한쪽 진영을 담당한다. **[결정됨]** A팀=왼쪽, B팀=오른쪽 (`lib/teams.js`). 좌/우 경계값은 50이며, 경계값(position === 50)은 오른쪽으로 취급한다.

## 경기(Match) 개념
- **[구현됨]** 전역적인 "현재 경기"라는 개념은 없다. 모든 위치 기록/점유율은 특정 경기(`matchId`)에 속하며, URL(`/matches/[id]`)이 어느 경기를 다루는지 나타낸다.
- 여러 경기가 동시에 `in_progress` 상태여도 된다(막지 않는다) — 서로 다른 탭/브라우저에서 여러 필드를 동시에 추적하는 사용 사례를 지원한다.
- 경기 상태는 `in_progress` → `ended`로만 전이한다(단방향, `ended`에서 되돌릴 수 없음).
- 경기가 `ended`면 해당 경기에는 더 이상 위치를 기록할 수 없지만(`POST /api/positions` 403), 이미 쌓인 기록/점유율은 그대로 조회할 수 있다.
- "리셋"(DB 전체 삭제) 기능은 없다. 새로 추적하고 싶으면 기존 경기를 종료하고 새 경기를 시작한다.
- **[구현됨]** `GET /api/matches`, `POST /api/matches`(생성), `PATCH /api/matches/:id`(종료, 이미 종료된 경기면 400)

## 데이터 입력 방식
- 공 위치 값은 수동으로 하나씩 입력한다.
- **[구현됨]** `POST /api/positions` — body `{ matchId, position: 0~100 정수 }`. 대상 경기가 없으면 400, 이미 종료됐으면(`ended`) 403. 성공 시 201과 저장된 레코드를 반환하며, 기록 시각(`recordedAt`)은 앱이 아닌 DB가 INSERT 시점에 채운다.
- **[구현됨]** `GET /api/positions?matchId=&limit=N` — `matchId` 필수(없으면 400). 해당 경기의 최근 기록 N건(기본 100, 최대 500)을 반환한다.
- **[구현됨]** 슬라이더 UI(`components/PositionForm.jsx`)가 3초마다 현재 값을 자동 전송한다(수동 제출 없음). 탭이 비활성화되면 전송을 일시정지하고, 다시 활성화되면 재개한다. "정지/재개"로 사용자가 직접 멈출 수도 있고, "경기 종료"로 해당 경기만 완전히 종료할 수 있다.

## 공격 점유율 정의
- 공격 점유율은 "공이 상대 진영에 머문 시간의 비율"로 정의한다.
- 즉 A팀 공격 점유율 = 공이 B팀 진영(A팀 기준 상대 진영)에 있었던 시간 / 전체 추적 시간
- **[구현됨]** `GET /api/occupancy?matchId=` — `matchId` 필수(없으면 400). 해당 경기의 기록만으로 각 팀의 공격 점유율(`attackRatio`)과 구간 시간을 반환한다. 기록이 2건 미만이거나 모든 timestamp가 동일하면 `insufficient_data` 상태를 반환한다.
- 점유율 계산은 연속된 두 기록 사이의 시간 구간만 누적하며, 마지막 기록 이후 구간(다음 기록이 없어 길이를 알 수 없는 구간)은 계산에서 제외한다.
- 점유 이벤트(누가 공을 소유했는지)는 이번 범위에 포함하지 않는다.

## 구현 상태
- DB 스키마(`migrations/`), 경기 생성/조회/종료 API, 경기별 위치 입력/조회 API, 경기별 점유율 조회 API, 경기 목록/추적 UI가 모두 구현되어 `main`에 반영되어 있다.
- 모든 페이지는 `getServerSideProps` 없이 클라이언트 사이드 `useEffect` + `fetch`로 데이터를 가져온다.
- 남은 후속 작업(문서·테스트 보강, 운영 안정화, 추가 기능 논의 등)은 `docs/plans/ball-position-tracking-work-plan.md`에서 단계별로 추적한다.
