# 요구사항: 축구공 위치 기반 점유율 추적

## 환경
- Node.js, Next.js
- MySQL (mysql2 라이브러리 사용)

## 도메인 규칙
- 공 위치는 1차원(좌/우)으로만 표현한다. 값은 0~100 사이의 정수이며, 0=완전 왼쪽, 100=완전 오른쪽을 의미한다.
- 팀은 두 개다: A팀(Anthro, 별칭 Ant), B팀(Bob, 별칭 Bike)
- 각 팀은 좌/우 중 한쪽 진영을 담당한다. **[결정됨]** A팀=왼쪽, B팀=오른쪽 (`lib/teams.js`). 좌/우 경계값은 50이며, 경계값(position === 50)은 오른쪽으로 취급한다.

## 데이터 입력 방식
- 공 위치 값은 수동으로 하나씩 입력한다.
- **[구현됨]** `POST /api/positions` — body `{ position: 0~100 정수 }`. 성공 시 201과 저장된 레코드를 반환하며, 기록 시각(`recordedAt`)은 앱이 아닌 DB가 INSERT 시점에 채운다.
- **[구현됨]** `GET /api/positions?limit=N` — 최근 기록 N건(기본 100, 최대 500)을 반환한다.
- **[구현됨]** 폼 UI(`components/PositionForm.jsx`)에서 0~100 숫자 입력으로 기록할 수 있다.

## 공격 점유율 정의
- 공격 점유율은 "공이 상대 진영에 머문 시간의 비율"로 정의한다.
- 즉 A팀 공격 점유율 = 공이 B팀 진영(A팀 기준 상대 진영)에 있었던 시간 / 전체 추적 시간
- **[구현됨]** `GET /api/occupancy` — 전체 기록을 기준으로 각 팀의 공격 점유율(`attackRatio`)과 구간 시간을 반환한다. 기록이 2건 미만이거나 모든 timestamp가 동일하면 `insufficient_data` 상태를 반환한다.
- 점유율 계산은 연속된 두 기록 사이의 시간 구간만 누적하며, 마지막 기록 이후 구간(다음 기록이 없어 길이를 알 수 없는 구간)은 계산에서 제외한다.
- 점유 이벤트(누가 공을 소유했는지)는 이번 범위에 포함하지 않는다.

## 구현 상태
- DB 스키마(`migrations/`), 위치 입력/조회 API, 점유율 조회 API, 입력 폼을 포함한 UI가 모두 구현되어 `main`에 반영되어 있다.
- 남은 후속 작업(문서·테스트 보강, 운영 안정화, 추가 기능 논의 등)은 `docs/plans/ball-position-tracking-work-plan.md`에서 단계별로 추적한다.
